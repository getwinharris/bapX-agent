# bapX Hermes Cleanup Spec

Remove ALL Hermes references from bapX codebase. bapX is 100% standalone.

## Rules:
- NO `.hermes` paths in code
- NO Hermes skills loading
- NO OAuth (ChatGPT, Claude, Nous, Qwen, GitHub Copilot auth)
- NO skills from Hermes loaded into UI
- NO codex from `~/.hermes/node/bin/codex` in bapX platform code
- Users bring their own API key only — single text field, pick provider from list

## Files to modify:

### 1. backend.py
- Remove: ALL_PROVIDERS OAuth entries (openai-oauth, nous-oauth, qwen-oauth, github-copilot) — keep only API key providers (openai, anthropic, google, deepseek, openrouter, xai, groq, mistral, together, cohere, perplexity, huggingface, minimax, kimi, xiaomi, zai, dashscope, stepfun)
- Remove: OAUTH_CONFIGS dict entirely
- Remove: SKILLS_DIR = HERMES_HOME / "skills"
- Remove: load_default_skills() function
- Remove: get_default_skills() function  
- Remove: DEFAULT_SKILLS_CACHE = None
- Remove: OAuthStartReq, OAuthPollReq pydantic models
- Remove: ALL routes: /api/auth/oauth/start, /api/auth/oauth/poll, /api/auth/oauth/status
- Remove: ALL routes: /api/skills, /api/user/skills (GET + POST)
- Simplify: /api/providers -> return only api_key providers (no oauth/copilot sections)
- Simplify: /api/user/api-key -> validate against only API key providers
- Simplify: /api/user/profile -> no oauth_tokens or skills_enabled fields
- Simplify: /api/agent/run -> remove skills loading (lines 767-772 area)
- Simplify: /api/chat/send -> remove OAuth credential fallback (use api_key only)
- Simplify: /health -> remove skill_count
- Simplify: /api/admin/automation/clear-cache -> remove DEFAULT_SKILLS_CACHE
- Simplify: /api/admin/users/{user_id} delete -> remove oauth_flows deletion
- Simplify: /api/admin/automation/cleanup-banned -> remove oauth_flows deletion
- Clean: Docstring at top — no "Hermes" references
- Remove: conn.execute tables for oauth_flows — keep table for backward compat but stop referencing it

- Remove: CODEX_CTO_BIN = ~/.hermes/node/bin/codex

- Remove: CODEX_API_KEY = hermes-codex-bridge env var — use a proper env variable or default

### 3. agent_runtime.py

- Remove: skills_enabled loading — no skills from Hermes
- Simplify to pure API-key-based provider resolution

- Remove: loadOAuthStatus(), renderOAuthProviders(), startOAuth(), showOAuthModal()
- Remove: copyOAuthCode(), cancelOAuth(), pollOAuth() functions

- Simplify: updateConnectionDisplay() — only check api_key, no OAuth logic

- Simplify: keyboard shortcut — remove cancelOAuth() from Escape handler
- Remove: oauth-grid CSS class
- Remove: .oauth-card CSS class
- Keep: skill-card CSS (repurpose for later user-created skills display)

Apply changes in order: backend.py first, then agent_runtime.py, agent_orchestrator.py, then dashboard.html, then index.html.

For each change, use patch() to find unique strings and replace/remove them. Read surrounding context if needed for uniqueness.
